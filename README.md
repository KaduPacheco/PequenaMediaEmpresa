# Sistema de Ponto Eletrônico - Landing Page

Esta é a landing page de um **Sistema de Ponto Eletrônico** focado em **Pequenas e Médias Empresas (PMEs)**. O projeto foi desenvolvido para apresentar os benefícios da solução, com foco em simplicidade, segurança jurídica e controle de jornada de forma eficiente.

## 💻 Sobre o Projeto

A landing page tem como objetivo captar empresas interessadas em uma solução digital para controle de ponto (batida de ponto). O design é voltado para a conversão, apresentando de maneira rápida e visual:
- Os problemas causados pelo controle manual.
- As soluções oferecidas pelo sistema.
- Os benefícios de produtividade e segurança.
- O plano de preços focado em custo-benefício (até 5 funcionários).
- Uma demonstração visual da interface do sistema real.

## 🚀 Tecnologias Utilizadas

O projeto foi construído utilizando tecnologias web padrão, sem dependência de frameworks complexos, garantindo alta performance e fácil manutenção:

- **HTML5**: Estruturação semântica da página.
- **CSS3**: Estilização visual, responsividade (Mobile First), variáveis CSS para o tema, e animações sutis (`fade-in`).
- **JavaScript (Vanilla)**: Interatividade leve, como animações baseadas no scroll utilizando `IntersectionObserver`.

## 📂 Estrutura de Arquivos

- `index.html`: Arquivo principal contendo toda a estrutura da landing page (Hero, Problemas, Soluções, Preço, Formulário, etc).
- `style.css`: Folha de estilos contendo o design system, tipografia (fonte Inter), cores, layouts em grid e e responsividade.
- `script.js`: Script responsável por criar as animações dos elementos quando eles entram no campo de visão (scroll).

## ⚙️ Como Executar

O projeto não necessita de um servidor de build ou dependências (como Node.js). Para visualizar a landing page, basta:

1. Clone este repositório:
   ```bash
   git clone https://github.com/KaduPacheco/PequenaMediaEmpresa.git
   ```
2. Abra a pasta do projeto.
3. Dê um duplo clique no arquivo `index.html` para abri-lo diretamente no seu navegador de preferência.

Alternativamente, você pode usar a extensão **Live Server** no VS Code para rodar com live-reload.

## ⚡ Estrutura da Landing Page

A página é dividida nas seguintes seções:
1. **Hero**: Promessa principal e CTA (Chamada de Ação).
2. **Problema**: Identificação das dores da empresa.
3. **Solução**: Como o sistema resolve os problemas.
4. **Benefícios**: Os ganhos de se usar a ferramenta.
5. **Precificação**: Destaque para o plano acessível.
6. **Prova Social / Mockup**: Demonstração de que o sistema é real.
7. **Segurança**: Pilares de confiabilidade.
8. **Formulário**: Captação de leads.
